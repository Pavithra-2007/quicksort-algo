Recursive implementation
// Find k-th smallest in arr[lo..hi] function quickSelectRec(arr, lo, hi, k): if lo == hi: return arr[lo] // base case — single element pivot = choosePivot(arr, lo, hi) // Last / Random / Median-of-Three swap(arr, pivot, hi) // move pivot to end p = partition(arr, lo, hi) // Lomuto partition if k == p : return arr[k] // pivot IS the answer elif k < p : return quickSelectRec(arr, lo, p-1, k) else : return quickSelectRec(arr, p+1, hi, k) median(arr): k = ⌊arr.length / 2⌋ return quickSelectRec(arr, 0, arr.length-1, k)
Iterative (non-recursive) implementation
// Same result — while loop replaces the call stack function quickSelectIter(arr, k): lo = 0 hi = arr.length - 1 while lo < hi: p = partition(arr, lo, hi) if p == k : break elif p < k : lo = p + 1 else : hi = p - 1 return arr[k] // Lomuto partition — pivot at arr[hi] function partition(arr, lo, hi): pivot = arr[hi]; i = lo - 1 for j = lo to hi-1: if arr[j] <= pivot: i++ swap(arr, i, j) // grow left partition swap(arr, i+1, hi) // place pivot return i + 1 // Pivot strategies choosePivot(arr, lo, hi, mode): if mode == "last" : return hi if mode == "random" : return randomInt(lo, hi) if mode == "median3" : mid = ⌊(lo + hi) / 2⌋ return medianIndex(arr, lo, mid, hi)
Recursive vs Iterative
Recursive
Calls itself with a smaller subproblem. Elegant and readable. Uses O(log n) call-stack depth on average; O(n) worst case — potential stack overflow on large sorted inputs.
Iterative
Updates lo/hi pointers in a loop. Preferred in production — O(1) auxiliary space, no stack overflow risk on adversarial input. Both versions perform identical comparisons.

Recurrence Intuition
Each call partitions O(n) elements and recurses into only one half:

T(n) = T(n/2) + O(n) → O(n) (Master Theorem, case 3)
Compare with merge sort: T(n) = 2·T(n/2) + O(n) → O(n log n)
QuickSelect recurses into one branch, not both — that's the entire advantage.

Priority Queue Selection
⏱ TimeO(n log k)

💾 SpaceO(k)
Use when: k is small or you need the top-k elements, not just the k-th.
Advantage: Guaranteed O(n log k). Good for streaming data.
Disadvantage: Heap overhead per comparison; slower than QuickSelect for k ≈ n/2.
